# 과거 물류 데이터 요청서 (현업·IT 전달용)

> 연주(연속주조) 부문 슬래브 물류를 **가시화하고 병목을 분석**하기 위해, 과거 공정
> 이벤트 로그를 요청드립니다. 아래 표준 형식으로 추출해 주시면 곧바로 분석에
> 활용할 수 있습니다. 형식 정의는 시스템의 `src/data.py`(로더·검증)와 일치합니다.

## 1. 목적 · 활용

- **가시화**: 과거 데이터로 슬래브 물류 흐름 재현
- **병목 분석**: 설비별 체류시간·대기·WIP로 병목 지점 식별
- **원인 분석 / 개선**: 병목 원인(설비 다운·전환 셋업·상류 변동) 진단 및 개선안 도출

> 개인식별정보(PII)는 필요하지 않습니다. 작업자 ID 등이 포함될 경우 마스킹/제외해 주세요.

## 2. 요청 데이터 개요

- **형태**: **공정 이벤트 로그** — 1행 = "heat(차지) 1개가 공정 1곳을 통과한 기록"
  (각 공정 **진입 시각**과 **이탈 시각** 포함)
- **대상 라인/설비**: 〔 예: 제2제강 연주 1~3호기 〕
- **대상 기간**: 〔 예: 최근 3~6개월(계절/제품믹스 변동 포함 권장) 〕
- **포함 범위**: 정상 흐름 + **지연/설비 다운/대기 등 비정상 상황 포함**(원인 분석에 필수)

## 3. 표준 스키마 (컬럼 정의)

`필수`는 반드시, `선택`은 가능하면 포함해 주세요. (선택 항목이 분석 깊이를 좌우합니다.)

| 컬럼 | 구분 | 타입 | 설명 | 예시 |
|------|:----:|------|------|------|
| `heat_no` | **필수** | 문자 | 차지/heat 식별자(케이스 ID) | `H0001` |
| `grade` | **필수** | 문자 | 강종 코드 | `API5L` |
| `stage` | **필수** | 문자 | 공정 위치(설비 단계) 코드 — §5 참고 | `mold` 또는 `몰드` |
| `enter_time` | **필수** | 일시 | 해당 공정 **진입** 시각 | `2026-06-01 06:13:35` |
| `exit_time` | **필수** | 일시 | 해당 공정 **이탈** 시각 | `2026-06-01 06:36:12` |
| `width_mm` | 선택 | 정수 | 슬래브 폭(mm) | `1600` |
| `thickness_mm` | 선택 | 정수 | 슬래브 두께(mm) | `230` |
| `equipment` | 선택 | 문자 | 설비 호기 식별(동일 공정 내 설비 구분) | `MOLD-1` |
| `reason_code` | 선택 | 문자 | 지연/대기/정지 **사유 코드** | `설비점검` |
| `due_date` | 선택 | 일시 | 납기/목표 시각 | `2026-06-01 14:00` |

> `enter_time`~`exit_time` 사이가 그 공정의 **체류시간(대기+처리)** 이 됩니다.
> 가능하면 **처리 시작 시각**(대기와 처리를 구분)도 별도 컬럼(`start_time`)으로
> 주시면 blocking(하류 막힘)·starving(상류 공급부족) 구분에 큰 도움이 됩니다.

### 최소 필수 세트 (전체가 어려울 경우)
`heat_no, grade, stage, enter_time, exit_time` 5개만 있어도 **가시화·병목 식별**은
바로 가능합니다. 선택 컬럼은 이후 추가 제공도 환영합니다.

## 4. 파일 형식 요구사항

- **파일 형식**: CSV (쉼표 구분), **UTF-8** 인코딩 (또는 Excel .xlsx도 가능)
- **날짜/시간**: `YYYY-MM-DD HH:MM:SS` (ISO 8601) 권장. 타임존/시각 동기화 명시
- **1행 = 1 공정 방문**, 헤더(컬럼명) 1행 포함
- 대용량이면 **월별 분할** 또는 압축(zip) 전달
- 결측값은 빈칸으로(임의 문자열/0 대체 금지)

분석 시스템의 표준 공정은 아래와 같습니다. 귀사의 실제 공정 단계/코드가 다르면,
**실제 코드 목록과 아래 표준으로의 매핑**을 함께 주세요. (더 세분화된 단계도 환영)

| 표준 stage | 라벨 | 설명 |
|-----------|------|------|

## 6. 코드북(데이터 사전) 요청

원인 분석의 정확도를 위해 아래 **코드 정의**를 함께 주세요.

- [ ] **강종 코드** 목록·설명 (grade)
- [ ] **사유 코드** 목록·설명 (reason_code) — 지연/대기/정지 유형
- [ ] **설비 목록** (equipment) — 공정별 설비 호기
- [ ] 공정(stage) 코드 목록·설명 (§5 매핑)

## 7. 데이터 품질 요청

- 각 heat는 **투입~완료까지 공정 이벤트가 누락 없이** 이어질 것
- 모든 행에서 `enter_time ≤ exit_time`
- heat/설비/강종 **식별자 표기 일관성**(같은 대상은 같은 코드)
- 시스템·설비 간 **시각 동기화** 확인

### 부록 — 자주 있는 질문

- **Q. 진입/이탈 시각이 아니라 이벤트 타임스탬프만 있습니다.**
  → heat·stage·timestamp·이벤트유형(진입/이탈) 형태여도 됩니다. 매핑해 변환하겠습니다.
- **Q. 폭/두께/납기는 다른 시스템(주문/설계)에 있습니다.**
  → heat_no로 조인 가능하면 별도 파일로 주셔도 됩니다.
- **Q. 데이터가 매우 큽니다.**
  → 우선 **1개월 표본**으로 형식을 맞춘 뒤 전체를 요청드리겠습니다.
