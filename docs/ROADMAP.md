# 로드맵 & 개선 백로그

현재 `cpn_slab.html`은 **개념 검증용 시각화 프로토타입**입니다.
아래는 알려진 한계와 향후 개선 방향을 우선순위별로 정리한 백로그입니다.
작업 시 이 문서를 갱신해 주세요.

---

## 알려진 한계 (현재 상태)

| # | 한계 | 위치/원인 |
|---|------|-----------|
| L1 | 실제 CPN 실행 엔진이 아님 — 토큰 이동이 확률 기반 | `tick()`의 `Math.random()` |
| L2 | Guard가 규칙으로 강제되지 않고 고정 시퀀스를 따름 | `stagesFor()` |
| L3 | 강종 개수 `3`이 하드코딩됨 | `tick()`의 `Math.random()*3` |
| L4 | 자동화 테스트/검증 없음 | 저장소 전반 |
| L5 | 실공정 데이터(MES/제조실행시스템) 연동 없음 | — |
| L6 | 상태가 메모리에만 존재 — 새로고침 시 소실 | — |
| L7 | 반응형 레이아웃 미흡 (좁은 화면에서 다이어그램 잘림) | CSS 고정폭 |
| L8 | 온도가 상태가 아닌 Place 기준 재계산값 | `makeTemp()` |

## 개선 백로그

### 우선순위: 높음

- **[P1] CPN 시맨틱 정식화**
  Place의 마킹(marking), Transition 발화(firing) 규칙, Arc/Guard 표현식을
  실제 데이터 구조로 모델링. `tick()`의 확률 기반 이동을 "발화 가능한
  Transition 탐색 → 발화" 방식으로 대체. (L1, L2 해소)

- **[P2] 하드코딩 제거**
  `Math.random()*3` → `GRADES.length`, 최대 슬래브·확률·병목 임계 등
  매직넘버를 상단 `CONFIG` 객체로 추출. (L3)

- **[P3] 기본 검증 자동화**
  최소한 헤드리스 브라우저(Playwright)로 "시작→투입→완료→초기화"
  스모크 테스트 추가. 콘솔 에러 0 확인. (L4)

### 우선순위: 중간

- **[P4] 실데이터 연동 인터페이스**
  Heat 스케줄/강종/설비 상태를 외부 소스(JSON/WebSocket/MES API)에서
  주입할 수 있는 어댑터 계층 설계. 시뮬레이션 모드 ↔ 실데이터 모드 토글.
  (L5)

- **[P5] 상태 영속화**
  진행 상태를 `localStorage` 또는 URL 파라미터로 저장/복원. (L6)

- **[P6] 반응형/모바일 레이아웃**
  SVG `viewBox` 스케일링 + 그리드 재배치로 좁은 화면 대응. (L7)

- **[P7] KPI 확장**
  Place별 대기시간, 강종별 리드타임, 처리량(TPH), 설비 가동률 등
  연주 부문에서 의미 있는 지표 추가.

### 우선순위: 낮음 / 장기

- **[P8] 코드 구조 분리**
  프로토타입이 커지면 단일 HTML을 `index.html` + `model.js` + `sim.js` +
  `render.js` 등으로 분리 (의존성 0 원칙은 유지, 번들러 도입은 별도 결정).

- **[P9] 시나리오 저장/재생**
  특정 투입 시퀀스를 저장해 재현 가능한 데모/디버깅 시나리오 구성.

- **[P10] 접근성**
  키보드 조작, ARIA 레이블, 색맹 대응(강종 구분을 색+패턴 병행).
